# 🚀 TeleMirror - Швидкий деплой на VPS

## 📋 Передумови

Для запуску вам потрібно:

1. **VPS сервер** з Ubuntu/Debian
2. **SSH доступ** до VPS
3. **Telegram API дані**:
   - API_ID і API_HASH з https://my.telegram.org/apps
   - SESSION_STRING (отримайте через `python login.py`)
4. **ID каналів** для дзеркалення

## 🎯 Швидкий старт

### Варіант 1: Інтерактивний деплой (рекомендовано)

```bash
# Зробити скрипт виконуваним
chmod +x quick-deploy.sh

# Запустити деплой
./quick-deploy.sh
```

Скрипт сам запитає всі необхідні дані та виконає деплой.

### Варіант 2: Ручне налаштування

1. **Створіть .env файл** з вашими даними:
```bash
cp .env-deploy .env
# Відредагуйте .env файл з вашими даними
```

2. **Запустіть традиційний деплой**:
```bash
./deploy.sh YOUR_VPS_IP root ~/.ssh/id_rsa
```

## 🔧 Необхідні дані

### Telegram API
- **API_ID** і **API_HASH**: отримайте на https://my.telegram.org/apps
- **SESSION_STRING**: запустіть `python login.py` і скопіюйте результат

### ID каналів
Щоб отримати ID каналу:
1. Додайте бота @userinfobot до вашого каналу
2. Бот надішле ID каналу у форматі `-100XXXXXXXXX`

### SSH доступ
- IP адреса вашого VPS
- SSH ключ (зазвичай `~/.ssh/id_rsa`)
- Користувач SSH (зазвичай `root`)

## 🌐 Після деплою

Після успішного деплою ваша аплікація буде доступна за адресою:
```
http://YOUR_VPS_IP:8000
```

## 🛠️ Корисні команди

### Переглянути логи
```bash
ssh -i ~/.ssh/id_rsa root@YOUR_VPS_IP 'cd /home/root/telemirror && docker-compose logs -f'
```

### Перевірити статус
```bash
ssh -i ~/.ssh/id_rsa root@YOUR_VPS_IP 'cd /home/root/telemirror && docker-compose ps'
```

### Перезапустити сервіс
```bash
ssh -i ~/.ssh/id_rsa root@YOUR_VPS_IP 'cd /home/root/telemirror && docker-compose restart'
```

### Оновити додаток
```bash
./quick-deploy.sh
```

## 📊 Контейнери

Деплой включає:
- **PostgreSQL** база даних (порт 5432)
- **TeleMirror** додаток (порт 8000)

## 🔒 Безпека

- Всі порти окрім 8000 закриті файрволом
- База даних доступна тільки для додатку
- SSH доступ через ключі

## 🔍 Діагностика

Якщо щось не працює:

1. **Перевірте SSH підключення**:
   ```bash
   ssh -i ~/.ssh/id_rsa root@YOUR_VPS_IP
   ```

2. **Перевірте статус контейнерів**:
   ```bash
   docker-compose ps
   ```

3. **Переглянути логи**:
   ```bash
   docker-compose logs -f telemirror
   ```

## 🆘 Підтримка

Якщо виникли проблеми:
1. Перевірте логи контейнерів
2. Переконайтесь, що всі дані в .env файлі коректні
3. Перевірте SSH підключення до VPS

---

**Примітка**: Цей скрипт автоматично встановить Docker і Docker Compose якщо вони не встановлені на вашому VPS.
